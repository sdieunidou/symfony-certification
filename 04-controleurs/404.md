# Générer des pages 404

## Concept clé
Lorsqu'une ressource n'est pas trouvée, le contrôleur doit déclencher une erreur 404 (Not Found).

## Application dans Symfony 7.0
On lance une exception qui implémente `HttpExceptionInterface` avec le code 404. Symfony fournit un helper.

## Exemple de code

```php
<?php

use App\Repository\ProductRepository;

public function show(int $id, ProductRepository $repo): Response
{
    $product = $repo->find($id);
    
    if (!$product) {
        // Helper AbstractController
        throw $this->createNotFoundException('Le produit n\'existe pas.');
    }
    
    // ...
}
```

## Points de vigilance (Certification)
*   **Exception** : C'est bien une **Exception** qui est lancée (`throw`), pas une réponse retournée. Cela permet d'interrompre l'exécution immédiatement.
*   **ParamConverter** : Si vous utilisez l'injection d'entité (`public function show(Product $product)`), Symfony génère **automatiquement** une 404 si l'entité n'est pas trouvée.
